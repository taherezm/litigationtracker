# AI/IP Litigation Tracker

Automated data pipeline for the IU IP & Technology Law Society AI/IP litigation tracker. The pipeline discovers newly filed AI/IP federal cases, monitors active dockets, summarizes new entries, and pushes the resulting JSON feed into `taherezm/iptl-iu-site`.

## Required GitHub Secrets

Add these repository secrets in `taherezm/ip-litigation-tracker` under Settings -> Secrets and variables -> Actions:

- `COURTLISTENER_API_KEY`: CourtListener API token.
- `ANTHROPIC_API_KEY`: Anthropic API key for relevance filtering and docket summaries.
- `IPTL_SITE_TOKEN`: GitHub personal access token with access to `taherezm/iptl-iu-site`.

## Creating `IPTL_SITE_TOKEN`

1. Open GitHub Settings.
2. Go to Developer Settings -> Personal Access Tokens -> Tokens (classic).
3. Create a token for the tracker workflow.
4. Check the `repo` scope.
5. Copy the token and save it as the `IPTL_SITE_TOKEN` secret in this repository.

## Running the Workflow Manually

1. Open the Actions tab in `taherezm/ip-litigation-tracker`.
2. Select `Litigation Tracker — Scheduled Update`.
3. Click `Run workflow`.

The workflow also runs automatically every five days at 13:00 UTC.

## Local Development

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the pipeline locally with the required environment variables set:

```bash
export COURTLISTENER_API_KEY=...
export ANTHROPIC_API_KEY=...
python scripts/discover_cases.py
python scripts/update_dockets.py
python scripts/summarize.py
```

`data/cases.json`, `data/updates.json`, and `data/last_run.json` are generated by the scripts. Do not hardcode API keys or manually enter routine updates.

## Adding a Case Manually if Needed

Manual additions should be rare. If a case needs to be seeded before the automation finds it, edit `data/cases.json` in the pipeline repository, follow the existing schema, and commit the change. The next workflow run will continue updating that case as long as it has `status: "active"` and a `courtlistener_docket_id`.
